REST API를 이용한 게시판 만들기
- 표현적 상태 전송 (Representational State Transfer)
- 2000년 대 초에 박사학위 논문
	현재 사용중인  URL 표현 방식이 과연 효과적인가?
	고객 목록을 가져오기 위한 http 링크는?
		/member/list
		/customer/list
		/client/customers
		/customer/data/list
	회원 등록을 위한 http링크
		/member/regist
		/member/add
	수정
		/member/edit
		/member/update
	삭제
		/member/delete
		/member/del

	결론) url에 대한 표준이 없다
	현재 http url 방식은 동사위주로 표현하고 있다 <- 문제점
	따라서 명사로 표시하기로 했다 (동사적 표현 -> 명사적 표현)
	구분은 HTTP의 전송 method를 사용해서 한다
		목록 : /member (GET)
		등록 : /member (POST)
		수정 : /member (PUT)
		삭제 : /member/23 (DELETE)
	ex) 게시판
		가져오기 : /board (GET)
		등록하기 : /board (POST)
		수정하기 : /board (PUT)
		삭제하기 : /board/45 (DELETE)
	Spring은 위의 url의 인식한다 == Spring은 'RESTful'하다
