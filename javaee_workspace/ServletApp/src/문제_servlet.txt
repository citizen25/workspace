HttpServlet 클래스의 생명주기에 대한 설명 중 틀린 것은?
(1) 개발자에 의해 작성된 서블릿을 서버에 배포해 놓으면 최초에 요청을 시도하는 클라이언트에 의해
   인스턴스가 생성되며, 이때 서블릿의 초기화도 함께 진행되는데, 이 초기화를 담당하는 생명주기
   메서드가 init() 이다  O
(2)일단 서블릿의 인스턴스가 생성된 후에는 클라이언트의 요청이 있더라도 인스턴스는 생성되지
않으며, 대신 쓰레드가 생성되어 요청을 처리하게 되는데, 이때 요청을 처리하기 위한 생명주기
메서드가 바로 service() 메서드 이다  O
(3)service() 메서드가 요청을 처리하기 위해서는 클라이언트가 요청 시 어떤 정보를 가지고 요청을 
했는지 알아야 하므로 요청 정보를 담고 있는 객체인 HttpServletRequest를 인수로 받도록 API가
정해져 있다.  O
(4)하나의 요청이 완료되려면, 들어온 요청에 대해 응답을 완료하는 시점까지를 의미하므로 service()
 메서드가 요청을 완료하려면 요청 객체인 HttpServletRequest 뿐만 아니라 응답 정보를 가진 객체
인 HttpServletResponse까지 인수로 받도록 API 정해져 있다. 결국 service() 메서드는 두개의 인수
를 받는 생명주기 메서드이다.  O
(5)클라이언트의 요청이 Get방식일 경우 doGet(), Post 방식일 경우엔 doPost()가 service() 메서드를
    호출한다.  X
	service가 doXXX를 호출한다. 반대
